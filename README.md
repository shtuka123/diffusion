# DiT From Scratch

A Diffusion Transformer (DiT) image generator written from the ground up in
CUDA and C++. No PyTorch at runtime. No libtorch, no cuDNN. Only cuBLAS for
matmul and cuRAND for random number generation; everything else — LayerNorm,
softmax, attention (forward and backward), optimizer steps, data loaders,
diffusion samplers, the DiT architecture itself — is hand-written from
scratch.

The network trains to generate class-conditional MNIST digits with
classifier-free guidance, and scales up (less convincingly) to generate
CIFAR-10 samples. This is a learning project: the goal was to understand
every line of code in a modern generative model by writing it rather than
importing it.

## Design principles

Six non-negotiable rules that held throughout:

1. **PyTorch is the oracle, not the crutch.** Every CUDA operation is
   compared against a PyTorch reference. No op is built upon until validated.
2. **Gradient checking is mandatory.** Every backward kernel is validated
   against numerical gradients on a tiny input before integration.
3. **One new thing at a time.** Commit after every passing test.
4. **FP32 throughout.** No mixed precision. Simpler, easier to debug.
5. **NaN is a real bug.** Never restart with a lower learning rate to make
   NaNs "go away."
6. **Only cuBLAS and cuRAND.** Everything else hand-written.

## Architecture overview

The forward pass, at the highest level:

    Input image (B, C, H, W)
        │
        ▼
      Patchify + linear projection  ──►  (B, T, D)   where T = (H/P) × (W/P)
        │
        │  + 2D positional embedding
        ▼
      DiT Block × N     (each block takes x and the conditioning vector c)
        │
        ▼
      Final adaLN + linear to patch output
        │
        ▼
      Unpatchify → predicted noise (B, C, H, W)

Each DiT block:

    x ──► adaLN₁(shift, scale from c) ──► MHA ──► × gate₁ (from c) ──► + residual ──► x_mid
    x_mid ──► adaLN₂(shift, scale from c) ──► MLP ──► × gate₂ (from c) ──► + residual ──► out

The conditioning vector `c = timestep_embed + class_embed` is fed into each
block, where a per-block modulation MLP produces six vectors: two shift/scale
pairs (for adaLN₁ and adaLN₂) and two gate scalars (for the residual branches).

Key architectural decisions (all from Peebles & Xie, 2022):

- **adaLN-Zero conditioning** instead of cross-attention or in-context
  tokens. Per-block modulation MLPs are initialized to zero, so each block
  starts as an identity map and only deviates as training signal demands.
  Critical for stability.
- **Pre-LN everywhere** (within adaLN), post-residual gates.
- **Learned 2D positional embeddings**, not sinusoidal. Simpler and works
  fine at MNIST/CIFAR scale.
- **Sinusoidal timestep embedding** through a small MLP to the conditioning
  dim.
- **Classifier-free guidance** via random label dropout (p=0.1) during
  training; at sampling, weighted extrapolation between conditional and
  unconditional predictions.

## Build

### Prerequisites

- **Windows 10/11** with Visual Studio 2022 (Desktop C++ workload), or
  **Linux / WSL2** with a recent GCC
- **CUDA Toolkit 13.1** or compatible
- **Python 3.10+** with PyTorch (CPU is fine — reference only)
- An NVIDIA GPU, compute capability ≥ 7.0. Edit `CMAKE_CUDA_ARCHITECTURES`
  in `CMakeLists.txt` to match yours.

### Building

    cmake -S . -B build -G Ninja
    cmake --build build -j

On Windows, use the included `dev_env.bat` to set up the MSVC environment
before running `cmake`. On Linux/WSL2, no extra setup needed.

### Reference data

Most tests depend on reference tensors generated by PyTorch. Generate them
once:

    python python_ref/gen_all_refs.py

This populates `data/ref/` with reference inputs and outputs for every
kernel's validation test.

### Running tests

    ./build/test_tensor                # Tensor RAII smoke tests
    ./build/test_matmul_vs_ref         # matmul vs PyTorch
    ./build/test_softmax_vs_ref        # numerically-stable softmax
    ./build/test_layernorm_vs_ref      # LayerNorm forward + backward
    ./build/test_attention_vs_ref      # full multi-head attention
    ./build/test_adaln_vs_ref          # adaLN-Zero
    ./build/test_gpt_tiny_vs_ref       # Phase 2 end-to-end
    ./build/test_diffusion_vs_ref      # Phase 3 end-to-end
    ./build/test_dit_vs_ref            # Phase 4 end-to-end
    ./build/grad_check_all             # numerical gradient checks on every backward

### Training

    # Phase 1 — MNIST classifier
    ./build/train_mnist_classifier

    # Phase 2 — GPT-tiny on tiny-shakespeare
    ./build/train_gpt_tiny

    # Phase 3 — MLP-denoiser diffusion on MNIST
    ./build/train_diffusion_mlp

    # Phase 4 — The real DiT
    ./build/train_dit_mnist
    ./build/train_dit_cifar    # stretch

Each writes checkpoints to `checkpoints/` and sample grids to `samples/` at
regular intervals. Hyperparameters are at the top of each training binary's
source file.

### Sampling

    # Generate a grid of 100 conditional MNIST samples
    ./build/sample_dit_mnist --cfg 3.0 --steps 50 --output samples/my_grid.png

    # Generate 10 samples of a specific class
    ./build/sample_dit_mnist --class 7 --n 10 --output samples/sevens.png

## Binary tensor format

Both Python and C++ use this format for all intermediate tensors:

    [4 bytes]            magic: 0x544E5352 ('TNSR', little-endian)
    [4 bytes]            ndim (uint32)
    [4 bytes × ndim]     shape (uint32 per dim)
    [4 bytes × numel]    data (float32, row-major)

Python implementation: `python_ref/tensor_io.py`. C++ implementation:
`include/tensor_io.h`.

## Known limitations and honest caveats

- **CIFAR-10 samples are not photorealistic.** A 15M-parameter DiT trained
  for a day on a consumer GPU can't match a 3B-parameter diffusion model
  trained on a datacenter for a month. The samples demonstrate that the
  architecture works, not that it beats the state of the art.
- **FP32 only.** Real production DiT work uses BF16 or mixed precision
  for 2–3x speedup. Adding mixed precision would be a separate project.
- **No gradient checkpointing, no Flash Attention, no fused kernels.**
  Attention is implemented as separate matmul + softmax + matmul kernels.
  Training is therefore 2–5x slower than a performance-optimized
  implementation would be.
- **Single-GPU only.** No DDP, no tensor parallelism.
- **Windows-specific build quirks** around MSVC version requirements. See
  the build docs.
